编译ffmpeg

Created2023-09-17|Updated2023-09-17|音视频学习
|Post View:

目标

  • 此前用brew安装的ffmpeg、ffprobe、ffplay,这里希望通过编译源码得到，主要为了实现下面的效果
    1. 编译出ffmpeg、ffprobe、ffplay三个命令行工具
    2. 只产生动态库，不产生静态库
    3. 将fdk-aac、x264、x265集成到FFmpeg中
      • x264、x265会在以后讲解的视频模块中用到

源码的下载

官网下载release包的源码（注意一定要release版本的，git上的版本，不一定能编译）。这里现在的事ffmpeg-6.0.tar.xz版本

编译

依赖项

  • brew install yasm

    • ffmpeg的编译过程依赖yasm
    • 若未安装yasm会出现错误：nasm/yasm not found or too old. Use --disable-x86asm for a crippled build.

  • brew install sdl2

    • ffplay依赖于sdl2
    • 如果缺少sdl2，就无法编译出ffplay

  • brew install fdk-aac

    • 不然会出现错误：ERROR: libfdk_aac not found

  • brew install x264

    • 不然会出现错误：ERROR: libx264 not found

  • brew install x265

    • 不然会出现错误：ERROR: libx265 not found

  • 注意，可以通过brew list的结果查看是否安装过,如果已经安装过，可以不用再执行brew install

    • brew list | grep fdk
    • brew list | grep x26
    • brew list | grep -E 'fdk|x26'

configure

进入源码目录，目录下有个configure可执行文件，通过该可执行文件生成makefile文件

  • 执行命令

    • --prefix

      • 用以指定编译好的FFmpeg安装到哪个目录
      • 一般放到/usr/local/ffmpeg中即可

    • --enable-shared

      • 生成动态库

    • --disable-static

      • 不生成静态库

    • --enable-libfdk-aac

      • 将fdk-aac内置到FFmpeg中

    • --enable-libx264

      • 将x264内置到FFmpeg中

    • --enable-libx265

      • 将x265内置到FFmpeg中

    • --enable-gpl

      • x264、x265要求开启GPL License

    • --enable-nonfree

      • fdk-aac与GPL不兼容，需要通过开启nonfree进行配置

编译和安装

  • 通过执行make -j8，开始编译代码，
    • -j8表示允许同时执行8个编译任务。
  • make install安装到之前指定的位置/usr/local/ffmpeg
  • 为了让bin目录中的ffmpeg、ffprobe、ffplay在任意位置都能够使用，需要先将bin目录配置到环境变量PATH中。

验证

  1. ffmpeg -version验证版本

  2. ffmpeg -codecs | grep aac，查询安装了的aac

  3. ffmpeg -codecs | grep x25，查询安装了的x264和x265
