存储属性

存储属性是一个作为特定类和结构体实例一部分的常量或变量。存储属性要么是变量存储属性 （由 var 关键字引入）要么是常量存储属性（由 let 关键字引入）

let 和 var 的区别：

  • 测试验证

从定义上

  • let:用来声明常量，常量的值一旦设置好便不能再被更改；
  • var:用来声明变量，变量的值可以在将来设置为不同的值。

汇编角度

  • 汇编查看
  • 两者并没有任何差别，无非把值赋值给对应的寄存器

SIL角度

  • swiftc xxx.swift -emit-sil // 生成sil（已优化

  • 生成的SIL

  • 存储属性在编译的时候，编译器默认会合成get/set方式，而我们访问/赋值存储属性的时候，实际上就是调用get/set。

  • let声明的属性默认不会提供setter,无法调用set方法，因此无法赋值

计算属性

类、结构体和枚举也能够定义计算属性，计算属性并不存储值，他们提供 getter 和 setter 来修改和获取值。

  • 测试验证

汇编角度

  • 汇编查看,实际会直接调用setter方法，采用直接派发静态调用的方式
  • setter方法

SIL角度

  • swiftc xxx.swift -emit-sil

  • 生成的SIL

  • 计算属性非@_hasStorage属性，但是具有getter和sertter方法，不占用实例的内存

属性观察器

  • 属性观察者会观察用来观察属性值的变化：
    • willSet 当属性将被改变调用，即使这个值与 原有的值相同；
    • didSet 在属性已经改变之后调用。
  • 测试验证

汇编角度

  • 调用初始化方式

  • 修改值,调用setter方法

  • willSet方法,这里再对radius进行修改，不会在触发setter方法(因此也不会触发属性观察器)，是直接赋值

  • didSetter方法,这里再对radius进行修改，不会在触发setter方法，是直接赋值

  • 注意,willSet、didSetter方法中不会触发，其对于属性的观察器，但是在设置其他属性的，同样会触发属性观察器,下面这种写法会陷入死循环

重写父类观察器

和重新方法逻辑一样,其实际调用的getter和setter是父类的方式，只是重写了属性观察器

  • 调用顺序是
    1. 子类will set
    2. 父类will set
    3. 设置存储值
    4. 父类did set
    5. 子类did set

汇编查看

  • 验证源码
  • 汇编查看子类setter，方法中会调用父类的的setter方法，因此会触发父类的设置的属性观察器

延迟存储属性

  • 必须有初始值

  • 延迟存储属性的初始值是在它第一次被使用时才进行计算

  • 用关键字lazy来标识一个延迟存储属性

  • 测试验证

SIL查看

  • swiftc xxx.swift -emit-sil

  • 生成的SIL

  • 懒加载属性第一次访问前没有值，访问后就有值，其本质就是在编译后生成一个可选型存储属性

getter方法分析

  1. 在初始化的时候,$__lazy_storage_$_radius设置的nil
  2. 拿到生成的属性 $__lazy_storage_$_radius的地址，然后去switch：如果有值，走bb1的代码块；如果没有值走bb2代码块
  3. bb2没有加载过：去构建初始值，给到变量$$__lazy_storage_$_radius
  4. bb1已加载过：已经有值了，直接给他返回出去

类型属性

  • 类型属性本质上就是一个全局变量
  • 类型属性只会被初始化一次,（使用的时候）
  • 测试验证

SIL查看

  • swiftc xxx.swift -emit-silgen
  • 编译后的sil里多了两个全局变量
    • sil_global private @$s4main6CircleV6radius_Wz,token for radius
    • $s4main6CircleV6radiusSivpZ,static Circle.radius
  • main函数中，@$s4main6CircleV6radiusSivau函数做了一个全局变量内存地址的转换
  • 全局变量内存地址的转换的函数s4main6CircleV6radiusSivau,builtin "once"底层源码调用的是swift_once，其底层实际是dispatch_once，这里的token则是dispatch_once_t,这里返回全局变量的地址
  • s4main6CircleV6radius_WZ方法，由builtin "once"保证只执行一次。在该方法中对radius进行了赋值。并且该赋值操作只在使用的时候才会进行操作
