延迟存储属性

  • lazy属性必须是var，不能是let

    • 必须在实例的初始化方法完成之前就拥有值

  • 如果多条线程同时第一次访问lazy属性

    • 无法保证属性只被初始化1次

  • 当结构体包含一个延迟存储属性时，只有var才能访问延迟存储属性

    • 因为延迟属性初始化时需要改变结构体的内存

      属性观察器

  • wil1set会传递新值，默认叫newValue

  • didset会传递旧值，默认叫oldValue

  • 在初始化器中设置属性值不会触发wil1Set和didset

  • 在属性定义时设置初始值也不会触发wi11Set和didset

    • 属性定义是设置，和初始化器中设置，其源码实际是一样的

  • 属性观察器，计算属性的功能，同样可以应用在全局变量、局部变量身上

    inout探究

    此前对函数inout进行探究，其值传入的是地址。但和属性值关联起来使用，会有所不同

  • 例子,用Int类型，能在内存中直观看到传的值

    单纯存储属性调用

  • 纯属性调用,test(&s.width),很显然是拿到数据地址(width是第一个，它的地址，即函数地址，第一个存的8，第二个是2)，作为rdi参数，之后进行函数调用。

计算属性调用

  • 计算属性调用,test(&s.area)，输出getArea、设置Num、setArea。首先计算属性会通过get取值拿到局部变量之后，将局部变量地址传进test函数去设置，设置完毕之后，再调用set方法。
  • 汇编验证
      1. `movq   0xb9a3(%rip) , %rdi`、`movq   0xb9a4(%rip), %rsi`,`rdi`和`rsi`作为参数，这里将`width`和`side`地址上的存`8`和`2`分别进行赋值；作为后续调用`area.getter`的参数
  2. `rax`作为返回值，这里将get返回的结果赋值给，`-0x28(%rbp)`地址空间上,`rbp`是局部变量，代码在main函数中，是main函数的局部变量
  3. `leaq   -0x28(%rbp), %rdi`,将局部变量的地址赋予`rdi`作为函数参数，调用`test(inout Swift.Int)`方法
  4. ` movq   -0x28(%rbp), %rdi`,`leaq   0xb97f(%rip), %r13`,将3中生成的局部变量的值赋值`rdi`作为参数,`s`地址也作为参数，调用`area.setter`进行赋值

带属性观察器的存储属性调用

  • 存储属性调用,test(&s.side)，输出设置Num、willSet 20、didSet 2 20。先设置值，方法执行完毕后，才调用willSet和didSet
  • 汇编验证
    1. movq 0xb8eb(%rip), %rax,movq %rax, -0x28(%rbp),这里取出sid的值，取出来，将他赋值给局部变量 -0x28(%rbp)
    2. leaq -0x28(%rbp), %rdi,这里将局部变量的地址取出作为参数，传递给test函数
    3. movq -0x28(%rbp), %rdi,leaq 0xb8cb(%rip), %r13,这里将修改后的局部变量，以及s作为参数，调用side.setter
    4. side.setter方法
      1. 这里r13，rdi的值始终保持不变，一个是s，一个是新值，也作为willSet方法和didiSet方法的参数调用
      2. 在%rax, 0x8(%r13),didiSet方法之前，将rax的值。即-0x30(%rbp)内存中的值(最开始rdi赋值的)，进行了新值的替换，r13是s，其后8字节即side
  • 有属性观察器的存储属性，实现和计算属性一样，采用拷入拷出或者值结果调用的方式，目的是为了设置完属性后，可以触发属性观察器。本质上test函数内部，是不会触发属性观察器的。

inout总结

  • 如果实参有物理内存地址，且没有设置属性观察器
    • 直接将实参的内存地址传入函数（实参进行引用传递）
  • 如果实参是计算属性 或者 设置了属性观察器
    • 采取了Copy In Copy Out(拷入拷出)的做法
      • 调用该函数时 ，先复制实参的值，产生副本(局部变量),调用get（存储属性是直接取值，计算属性是get）
      • 将副本的内存地址传入函数（副本进行引1用传递），在函数内部可以修改副本的值
      • 函数返回后，再将副本的值覆盖实参的值,调用(set)
  • 总结：inout的本质就是引用传递（地址传递）

    类型属性

  • 严格来说，属性可以分为
    • 实例属性 ( Instance Property)：只能通过实例去访问
      • 存储实例属性 ( Stored Instance Property）：存储在实例的内存中，每个实例都有1份，
      • 计算实例属性 ( Computed Instance Property )
    • 类型属性( Type Property ）：只能通过类型去访问
      • 存储类型属性 ( Stored Type Property)：整个程序运行过程中，就只有1份内存（类似于全局变量)
      • 计算类型属性 ( Computed Type Property )

        类型属性细节

  • 不同于存储实例属性，你必须给存储类型属性设定初始值
    • 因为类型没有像实例那样的init初始化器来初始化存储属性
  • 存储类型属性默认就是lazy，会在第一次使用的时候才初始化
    • 就算被多个线程同时访问，保证只会初始化一次,会使用dispatch_once技术
    • 存储类型属性可以是let，因为是类型属性，不用遵循（实例变量初始化之前，let必须有值，因此在实例对象中let不能和lazy使用）
  • 枚举类型也可以定义类型属性（存储类型属性、计算类型属性）
    • 枚举类型中不可以定义存储实例属性，但是可以定义类型属性
    • 这是因为类型属性，不在实例中，属于全局变量

      单例模式

  1. private保证外界不会调用初始化方法
  2. 使用类型属性，是lazy，会在第一次使用的时候才初始化，并且保证只会初始化一次，线程安全的,和OC中dispatch_once
