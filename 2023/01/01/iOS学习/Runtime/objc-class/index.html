Class的结构

class_rw_t

class_rw_t里面的methods、properties、protocols是二维数组，是可读可写的，包含了类的初始内容、分类的内容。

class_ro_t

class_ro_t里面的baseMethodList、baseProtocols、ivars、baseProperties是一维数组，是只读的，包含了类的初始内容。

method_t

method_t是对方法\函数的封装

  • IMP代表函数的具体实现
    • typedef id _Nullable (*IMP)(id _Nonnull,SEL _Nonnull,...)
  • SEL代表方法\函数名，一般叫做选择器，底层结构跟char *类似
    • 可以通过@selector()和sel_registerName()获得
    • 可以通过sel_getName()和NSStringFromSelector()转成字符串
    • 不同类中相同名字的方法，所对应的方法选择器是相同的
  • types包含了函数返回值、参数编码的字符串
    • 方法为objc_msgSend(id self,SEL,...)
    • i->int,@->id,:->SEL,f->float
    • 24参数总字节，0从0字节开始，8从第8字节开始

方法缓存

Class内部结构中有个方法缓存（cache_t），用散列表（哈希表）来缓存曾经调用过的方法，可以提高方法的查找速度

  • mask存储的是散列表的长度减1
  • 缓存查找
    • objc-cache.mm
    • void cache_t::insert(SEL sel, IMP imp, id receiver)

源码分析

  1. 插入之前，会先判断是否需要扩容，扩容的条件为，大于之前的3/4倍,扩容为2倍扩容，不超过最大缓存尺寸(最大为MAX_CACHE_SIZE = 1<<16,最小为INIT_CACHE_SIZE = 1<<2)
  2. 当扩容时，生成的新的buckets会直接替换旧的，会清除之前缓存的方法，也就是说，之前缓存的方法，不会传给新扩容的缓存，缓存会全部释放。新缓存又要重新开始计数。
  3. 向缓存中插入方法
  4. hash的方式是直接用SEL & mask得到key的方式。存在冲突时，会根据取到的sel判断是不是同一方法
    • 解决hash冲突的方式，开放地址法-线性探测
